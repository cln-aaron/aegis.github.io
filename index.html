// SECTION 02 · ARCHITECTURE

Eight layers,
one AI-native data plane.

From discovery at the edge to continuous offence at the core — every layer feeds a single correlation engine and a single investigation console.

01

Continuous AI asset discovery

Real-time inventory of every AI-related asset across code, cloud, browsers, OAuth grants, network flows, and SaaS — sanctioned, in-house, or shadow.

  • Browser, OAuth, and network-based shadow AI discovery
  • Code and container scanning for AI frameworks and SDKs
  • Cloud-native AI service detection across AWS, Azure, GCP
  • MCP server and agent skill enumeration
  • Live dependency graph: which app uses which model, agent, and tool
02

Model & supply chain provenance

Every model and dataset entering the enterprise carries a cryptographically verifiable provenance record, enforced at ingestion and continuously re-verified.

  • Signature verification and model namespace-reuse detection
  • Pickle and safetensors deserialisation scanning (nullifAI class)
  • CycloneDX ML-BOM and SPDX 3.0.1 AI / Dataset Profile compliance
  • Dataset licence verification and provenance tracing
  • Continuous re-scan of deployed models against new CVEs and threat intel
03

Identity & access for AI

Every AI agent, MCP server, and inference call is a first-class non-human identity. Scoped, short-lived credentials. Just-in-time access. Tamper-evident receipts.

  • Non-human identity (NHI) governance for models, agents, MCP servers
  • Just-in-time scoped credentials, destroyed on session completion
  • Attribute- and intent-based access control for tool invocation
  • Native integration with CyberArk, Okta, Microsoft Entra, Saviynt
  • AARM-aligned session context evaluation, not just static permissions
04

AI security data lake

The architectural keystone: an open AI security telemetry schema capturing every prompt, completion, tool call, identity exchange, and inference event.

  • Open telemetry schema (OCSF extension proposal)
  • Prompts, completions, system-prompt state changes, tool invocations
  • Agent-to-agent (A2A) and MCP transaction capture
  • GPU and DPU-level inference event capture
  • RAG retrieval, embedding query, and vector store events
  • Pipes natively into Splunk, Sentinel, Elastic, Datadog, Sumo Logic
05

Correlation & the AI kill chain

AI-specific kill-chain detection logic that converts low-fidelity signals into high-fidelity incidents — surfacing multi-stage attacks single tools cannot see.

  • Prompt injection → unauthorised tool invocation chains
  • Compositional exfiltration: permitted actions that constitute breach
  • System prompt drift and tampering detection
  • Cross-session credential reuse by an agent identity
  • Membership inference and training-data extraction detection
  • Model namespace substitution between SDK pull and provenance
06

Investigation & response (AI-SOC)

An investigation console built for the AI-SOC analyst. Full session reconstruction, lineage views, and orchestrated AI-native response playbooks.

  • Full multi-turn session reconstruction across prompts, tools, identities
  • Lineage view: output → model → training data → retrieval source
  • Hallucination, jailbreak, and exfiltration triage workflows
  • One-click: revoke credentials, quarantine models, roll back prompts
  • Usage- and asset-based metering tied to the AI initiative budget
07

Drift, bias & in-house model assurance

MLOps-grade dashboards for teams building proprietary models — wired directly into the security correlation engine. Drift becomes a detection signal.

  • Model drift, data drift, and concept drift monitoring
  • Fairness, bias regression, and protected-attribute analysis
  • Hallucination rate tracking with confidence calibration
  • Toxic output and policy-violation rate dashboards
  • Bridge between MLOps notebooks and the SOC analyst's queue
08FLAGSHIP

Continuous AI red teaming

The differentiator. Red teaming moves from a procured report to a continuous, automated capability running inside the same data plane as your detections. Every finding tunes defences. Every defence improvement spawns new attacks.

  • Adversarial prompt generation vs. OWASP LLM Top 10 & refreshed jailbreak libs
  • Multi-turn simulation: indirect injection, confused-deputy, compositional exfil
  • Agent kill-chain execution against real agents, with safe-execution scaffolding
  • Supply chain simulation: namespace-reuse, pickle deser, MCP tool poisoning
  • Model robustness: adversarial examples, membership inference, model extraction
  • Purple-team workflow: every finding opens an incident in the same console
  • Compliance-aligned: NIST AI RMF, EU AI Act Annex IV, ISO/IEC 42001, OWASP

The defensive stack and the offensive stack become a single feedback loop — the flywheel no point vendor can build.
